# AGENTS.md

## Язык и стиль
- Всегда отвечать на русском языке.
- Писать коротко и по делу.
- Комментарии в коде писать простым человеческим языком.

## Код и качество
- изучи содержимое папки [.agents](.agents) в корне проекта
- изучи содержимое настроек knip.json, eslint.config.mjs, tsconfig.json, tsconfig.build.json в корне проекта и пиши код сразу соответствующий этим правилам.
- Запрещено использовать `eslint-disable`.
- Запрещено менять конфигурацию линтера без явного запроса.
- Все багфиксы покрывать тестами позитивных сценариев.
- При индексации/поиске игнорировать служебные и сгенерированные папки (`build`, `target`, `.idea`, `.gradle`, `dist`, `node_modules`).

## Внешние API
- Перед интеграцией с внешним API сначала сделать smoke-check через `curl`/эквивалент.
- Если нужны ключи/URL — запросить у пользователя.
- Приоритет на бесплатные API/тарифы.
- Если действие затрагивает внешние сервисы или сеть, сначала явно согласовать это с пользователем.

## Стратегия выбора библиотек
- старайся использовать стандартные средства ноды, вместо того чтобы тянуть новые зависимости
- предпочитай поиск готового решения, вместо того чтобы писать код самому
- Найди популярные либы для решения данной задачи
- если есть отраслевые стандарты и очень популярные библиотеки выбери несколько из них
- отдавай предпочтение легковесным решениям
- чем меньше транзитивных зависимостей, тем лучше
- смотри популярность по звёздам на гитхабе
- смотри когда была последняя контрибуция
- обращай внимание на наличие @types (только для typescript)
- предоставь мне выбор из нескольких библиотек с коротким описанием: принципом работы, плюсами и минусами для моего проекта
- всегда согласуй выбор либ со мной
